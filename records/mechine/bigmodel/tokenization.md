# 标记化
## 1. 作用
j

## n. 常用方法
1. 基于空格的分词
    方法: 按空格、标点等分隔符拆分文本。
    优点: 简单高效。
    缺点: 无法处理无空格语言（如中文）或复杂结构。

2. 基于规则的分词
方法: 使用预定义规则（如正则表达式）拆分文本。
优点: 可定制，适合特定领域。
缺点: 规则复杂且难以覆盖所有情况。

3. 基于字典的分词
方法: 利用词典匹配文本中的词。
优点: 适用于有明确词汇边界的语言。
缺点: 依赖词典质量，难以处理新词。

4. 基于统计的分词
方法: 使用统计模型（如n-gram、HMM、CRF）进行分词。
优点: 能处理未登录词，适应性强。
缺点: 需要大量标注数据，计算复杂。

5. 基于机器学习的分词
方法: 使用机器学习模型（如SVM、神经网络）进行分词。
优点: 灵活，适应不同语言和领域。
缺点: 依赖大量标注数据，训练时间长。

6. 基于深度学习的分词
方法: 使用深度学习模型（如LSTM、BERT）进行分词。
优点: 效果好，能捕捉复杂语言特征。
缺点: 计算资源需求高，训练时间长。

7. 子词标记化（Subword Tokenization）
方法: 将词拆分为更小的子词单元。
    常见算法:
    Byte Pair Encoding (BPE): 通过合并高频字节对生成子词。
    WordPiece: 类似BPE，基于概率模型。
    Unigram Language Model: 基于概率模型选择最优子词。
优点: 能处理未登录词，减少词汇表大小。
缺点: 可能生成不直观的子词。

8. 字符级标记化（Character-level Tokenization）
方法: 将文本拆分为单个字符。
优点: 简单，适用于所有语言。
缺点: 序列长度增加，计算复杂度高。

9. 混合标记化
方法: 结合多种标记化方法。
优点: 灵活，适应不同需求。
缺点: 实现复杂，需权衡不同方法。

10. 预训练模型中的标记化
方法: 使用预训练模型（如BERT、GPT）的标记化工具。
优点: 与模型兼容，效果好。
缺点: 依赖特定模型，灵活性低。
