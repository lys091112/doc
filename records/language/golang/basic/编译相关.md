# 文件命名和编译相关

## 1. 文件命令

### 1.1 平台区分

文件名_平台 ``file_windows.go, file_unix.go``

可选为：windows, unix, posix, plan9, darwin, bsd, linux, freebsd, nacl, netbsd, openbsd, solaris, dragonfly, bsd, notbsd， android，stubs

### 1.2 测试单元

文件名_test.go或者 文件名_平台_test.go ``path_test.go,  path_windows_test.go``

### 1.4 CPU类型区分, 汇编用的多

文件名_(平台:可选)_CPU类型.  ``vdso_linux_amd64.go``

可选：amd64, none, 386, arm, arm64, mips64, s390,mips64x,ppc64x, nonppc64x, s390x, x86,amd64p32

## 2. 平台编译命令详解

### 2.1 go build

1. ``go build`` 的时候会选择性地编译以系统名结尾的文件(linux、darwin、windows、freebsd)。例如Linux(Unix)系统下编译只会选择array_linux.go文件，其它系统命名后缀文件全部忽略。

2. 在xxx.go文件的文件头上添加 ``// + build !windows (tags)`` ，可以选择在windows系统下面不编译 
    ```go
    // +build !windows
    package main
    ```
3. 打包链接: 将go包打包成底层的链接包,给其他语言使用
```sh 
   go build -buildmode=c-archive -o output/macos/libadd.dylib add_lib.go
```

#### 2.1.1 平台打包编译示例
在linux平台上 可以首先使用uname -a 查看平台版本和架构
``` sh
GOOS=linux GOARCH=amd64 go build $(LD_FLAGS) -o $(BIN_DIR)/linux/$(APP_NAME) ./main.go
GOOS=linux GOARCH=arm64 go build $(LD_FLAGS) -o $(BIN_DIR)/linux/$(APP_NAME)-arm64 ./main.go

@mkdir -p $(BIN_DIR)/windows
GOOS=windows GOARCH=amd64 go build $(LD_FLAGS) -o $(BIN_DIR)/windows/$(APP_NAME).exe ./main.go

@mkdir -p $(BIN_DIR)/darwin
GOOS=darwin GOARCH=amd64 go build $(LD_FLAGS) -o $(BIN_DIR)/darwin/$(APP_NAME) ./main.go
GOOS=darwin GOARCH=arm64 go build $(LD_FLAGS) -o $(BIN_DIR)/darwin/$(APP_NAME)-arm64 ./main.go
```
